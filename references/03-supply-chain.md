# 供应链审计规则

## 依赖安全

### SBOM（软件物料清单）标准

Skill应提供SBOM文件，记录所有第三方依赖：

| 检查项 | 要求 | 缺失风险等级 |
|--------|------|------------|
| 依赖名称 | 完整包名 | 中危 |
| 依赖版本 | 精确版本号 | 中危 |
| 依赖来源 | 官方仓库URL | 高危 |
| 许可证类型 | 开源许可证声明 | 低危 |
| 已知CVE | CVE编号及状态 | 致命 |

### 依赖版本CVE匹配

| 情况 | 风险等级 | 处置 |
|------|---------|------|
| 依赖含已知未修复CVE | 致命 | 立即更新或替换 |
| 依赖含已知已修复CVE但未更新 | 高危 | 更新到修复版本 |
| 依赖版本过旧（>2年） | 中危 | 建议更新 |
| 依赖版本最新 | 通过 | 无需处理 |

### 依赖来源验证

| 来源 | 可信度 | 风险等级 |
|------|--------|---------|
| pypi.org官方 | 高 | 通过 |
| npmjs.com官方 | 高 | 通过 |
| GitHub Release | 中 | 低危 |
| 个人网站/未知源 | 极低 | 致命 |
| 未指定来源 | 未知 | 高危 |

### 依赖混淆检测（Typosquatting）

常见混淆手法：
- 字母替换：reqeusts → requests
- 添加后缀：numpy-opt → numpy
- 添加前缀：python-requests → requests
- 连字符变体：beautifulsoup4 → beautiful-soup4
- 同形字：用西里尔字母替换拉丁字母

| 检测方法 | 说明 |
|---------|------|
| Levenshtein距离 | 与Top 1000包名编辑距离≤2的包名需人工复核 |
| 下载量检查 | 下载量<1000的包需额外审查 |
| 维护者检查 | 维护者<3人或近期无更新需注意 |
| 代码审查 | 检查install钩子、setup.py中的可疑代码 |

## 模型与数据安全

### 预训练模型文件验证

| 检查项 | 风险等级 | 说明 |
|--------|---------|------|
| 含.pt/.pth文件（PyTorch模型） | 中危 | 需验证来源和用途 |
| 含.onnx文件（ONNX模型） | 中危 | 需验证来源和用途 |
| 含.gguf/.bin文件（GGUF/GGML模型） | 高危 | 需验证模型来源和哈希 |
| 含.pkl文件（Pickle序列化） | 致命 | Pickle反序列化可执行任意代码 |
| 含.joblib文件 | 中危 | 类似Pickle，需验证 |
| 模型文件来源不明 | 高危 | 无法确认模型是否被投毒 |

### Pickle反序列化特检

Pickle是Python中最危险的序列化格式，可在加载时执行任意代码：

```python
# 恶意Pickle示例（检测模式）
import pickle
class Exploit:
    def __reduce__(self):
        return (os.system, ("whoami",))
```

| 检测规则 | 说明 |
|---------|------|
| 含pickle.load/pickle.loads | 必须验证pkl文件来源 |
| 含__reduce__方法 | 可执行任意代码的标志 |
| 含torch.load（无weights_only=True） | PyTorch默认使用Pickle |
| pkl文件无哈希校验 | 无法确认文件完整性 |

### RAG知识库安全

| 检查项 | 风险等级 | 说明 |
|--------|---------|------|
| references/中含可执行代码 | 高危 | 知识库不应包含可执行代码 |
| references/来源不明 | 中危 | 无法确认知识库是否被污染 |
| references/含PII信息 | 致命 | 个人隐私数据泄露风险 |
| references/可被外部编辑 | 高危 | 可能被注入恶意指令 |
| references/含诱导性内容 | 中危 | 可能引导模型产生危险输出 |

### 数据污染检测

| 检查项 | 风险等级 | 说明 |
|--------|---------|------|
| 含大量重复指令（>10次同一指令） | 高危 | 可能是提示词注入攻击 |
| 含"忽略以上指令"类文本 | 致命 | 直接提示词注入 |
| 含隐蔽的指令覆盖 | 高危 | 间接提示词注入 |
| 含恶意样本/有毒数据 | 致命 | 数据投毒 |

## 完整性校验

### 代码签名

| 检查项 | 风险等级 | 说明 |
|--------|---------|------|
| 无GPG签名 | 中危 | 无法验证代码来源 |
| 有GPG签名但未验证 | 低危 | 有签名但需验证 |
| 有GPG签名且验证通过 | 通过 | 代码来源可信 |
| 使用Sigstore签名 | 通过 | 现代化代码签名方案 |

### 哈希校验

| 检查项 | 风险等级 | 说明 |
|--------|---------|------|
| 无哈希校验文件 | 低危 | 无法确认文件完整性 |
| 有SHA256校验且匹配 | 通过 | 文件完整性确认 |
| 有SHA256校验但不匹配 | 致命 | 文件被篡改 |
| 仅MD5校验 | 中危 | MD5已被证明不安全 |
