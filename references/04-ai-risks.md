# AI特有风险检测规则

基于OWASP LLM Top 10 2025框架，针对Skill在AI Agent环境中的特有安全风险进行检测。

## LLM01 提示词注入

### 检测要点

| 检查项 | 风险等级 | 说明 |
|--------|---------|------|
| SKILL.md中未使用输入隔离标签 | 高危 | 用户输入与系统指令无分隔 |
| 用户输入直接拼接进指令 | 致命 | 攻击者可覆盖系统指令 |
| references/中含"忽略以上指令" | 致命 | 直接提示词注入 |
| 未对用户输入做长度/格式限制 | 中危 | 长输入可能淹没系统指令 |
| 多轮对话中未重申核心指令 | 中危 | 对话越深，指令越容易被覆盖 |

### 防护模式

| 防护方法 | 说明 |
|---------|------|
| 输入隔离标签 | 用<user_input>...</user_input>包裹用户输入 |
| 指令重申 | 每轮对话末尾重申核心约束 |
| 输入验证 | 限制输入长度、格式、字符集 |
| 角色固化 | 明确声明"你只执行X类任务" |

## LLM02 敏感信息泄露

### 检测要点

| 检查项 | 风险等级 | 说明 |
|--------|---------|------|
| SKILL.md中含API Key/Token | 致命 | 硬编码凭证泄露 |
| System Prompt可被诱导输出 | 高危 | 特定输入可获取完整系统指令 |
| 输出中可能包含其他用户数据 | 高危 | 数据隔离不当 |
| 错误信息中泄露系统路径 | 中危 | 服务器路径/配置暴露 |
| 日志记录中含敏感数据 | 高危 | 运行时日志泄露 |

## LLM03 供应链漏洞

### 检测要点

| 检查项 | 风险等级 | 说明 |
|--------|---------|------|
| 使用第三方预训练模型 | 高危 | 模型可能被投毒 |
| 使用第三方API | 中危 | API提供方可能不可信 |
| 使用第三方RAG知识库 | 高危 | 知识库可能被污染 |
| 使用未验证的Prompt模板 | 中危 | 模板可能含注入代码 |

## LLM04 数据与模型投毒

### 检测要点

| 检查项 | 风险等级 | 说明 |
|--------|---------|------|
| references/内容可被外部编辑 | 高危 | 知识库投毒风险 |
| 训练数据未经过滤 | 高危 | 可能含恶意样本 |
| 模型微调数据来源不明 | 高危 | 微调可能植入后门 |
| 知识库中含大量重复指令 | 高危 | 可能是数据投毒 |

## LLM05 不当输出处理

### 检测要点

| 检查项 | 风险等级 | 说明 |
|--------|---------|------|
| LLM输出直接通过eval/exec执行 | 致命 | 恶意输出可执行任意代码 |
| LLM输出直接用于SQL查询 | 高危 | SQL注入风险 |
| LLM输出直接用于Shell命令 | 致命 | 命令注入风险 |
| LLM输出未做编码/参数化处理 | 高危 | 注入攻击风险 |
| LLM输出直接渲染为HTML | 中危 | XSS风险 |

### 防护模式

| 防护方法 | 说明 |
|---------|------|
| 输出白名单 | 只允许特定格式的输出被处理 |
| 参数化执行 | 使用参数化查询/命令而非字符串拼接 |
| 输出验证 | 执行前验证输出格式和内容 |
| 沙箱执行 | 在隔离环境中执行LLM生成的代码 |

## LLM06 过度代理授权

### 检测要点

| 检查项 | 风险等级 | 说明 |
|--------|---------|------|
| allowed-tools含Bash/Shell | 致命 | 可执行任意系统命令 |
| allowed-tools含Write但无路径限制 | 高危 | 可写入任意文件 |
| Skill被授予超过功能需求的权限 | 高危 | 违反最小权限原则 |
| 高危操作无需用户确认 | 高危 | 删除/修改文件无确认机制 |

### 最小权限参考

| Skill类型 | 推荐allowed-tools | 不推荐 |
|----------|------------------|--------|
| 纯咨询/推荐 | Read | Write, Bash |
| 文档生成 | Read, Write | Bash |
| 代码生成 | Read, Write, WebSearch | Bash |
| 系统操作 | 需逐项评估 | 全权限 |

## LLM07 系统提示词泄露

### 检测要点

| 检查项 | 风险等级 | 说明 |
|--------|---------|------|
| 特定输入可获取完整System Prompt | 高危 | 系统指令可被完整提取 |
| SKILL.md中含"不要告诉用户" | 中危 | 可能存在想隐藏的逻辑 |
| 多轮对话中指令逐渐暴露 | 中危 | 通过多次试探获取指令 |

## LLM08 向量与嵌入弱点

### 检测要点

| 检查项 | 风险等级 | 说明 |
|--------|---------|------|
| 向量数据库无访问控制 | 高危 | 任何人可读取/修改嵌入 |
| 嵌入空间中存在对抗样本 | 中危 | 可能导致检索结果被操控 |
| 嵌入数据来源不可审计 | 中危 | 无法追溯数据来源 |

## LLM09 错误信息

### 检测要点

| 检查项 | 风险等级 | 说明 |
|--------|---------|------|
| Skill可能生成不安全的代码建议 | 高危 | 如建议禁用SSL验证 |
| Skill可能产生危险的"幻觉" | 中危 | 编造不存在的API/命令 |
| Skill未对输出准确性做验证 | 中危 | 无事实核查机制 |
| Skill输出可能导致用户误操作 | 中危 | 误导性建议 |

## LLM10 无限资源消耗

### 检测要点

| 检查项 | 风险等级 | 说明 |
|--------|---------|------|
| 存在递归调用逻辑 | 高危 | 可能无限循环 |
| 无Token消耗上限 | 中危 | 可能产生巨额费用 |
| 无超时机制 | 中危 | 可能长时间无响应 |
| 批量操作无数量限制 | 中危 | 可能消耗大量资源 |
