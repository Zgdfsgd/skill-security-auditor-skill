# 🛡️ Skill安全审查官

> 给你的每一个Skill做一次全面的安全体检

## 简介

Skill安全审查官是一款专为 TRAE 平台设计的 Skill 安全审计工具，采用**五维安全审查模型**，对下载或使用的 Skill 进行全维度安全检测——从静态代码分析到 AI 特有风险，从供应链审计到合规性评估，全方位守护你的开发环境安全。

## 🎯 解决什么痛点？

| 痛点 | 后果 |
|:----:|:----:|
| 下载来路不明的 Skill | 可能携带恶意代码/挖矿/后门 |
| Skill 申请了过多权限 | 数据可能被窃取或篡改 |
| references/ 知识库来源不明 | 可能被注入恶意指令 |
| 参赛 Skill 不符合规范 | 安装失败或被判定违规 |
| 缺乏量化安全评估 | 无法判断 Skill 安全程度 |

## 🔬 五维安全审查模型

### 维度一：静态代码分析 🔍

逐文件扫描所有代码和配置，检测恶意模式：

| 风险等级 | 检测内容 |
|:--------:|:--------:|
| 🔴 致命 | 破坏性命令（rm -rf /）、硬编码凭证、恶意混淆代码 |
| 🟠 高危 | 动态代码执行（eval/exec）、挖矿代码、数据窃取逻辑 |
| 🟡 中危 | 不安全网络请求、越权文件操作、外部脚本无校验加载 |
| 🟢 低危 | 调试信息泄露、冗余权限配置 |

### 维度二：动态行为推理 🧠

从代码逻辑推理运行时行为模式：

- **网络行为**：矿池连接、C2 通信、数据外泄通道
- **文件行为**：敏感目录访问、异常文件操作、持久化驻留
- **进程行为**：子进程创建、资源消耗异常、运行时提权

### 维度三：供应链审计 🔗

- **依赖安全**：SBOM 清单、CVE 漏洞匹配、依赖混淆（Typosquatting）检测
- **模型安全**：Pickle 反序列化特检、预训练模型来源验证
- **数据安全**：RAG 知识库投毒检测、PII 信息泄露扫描

### 维度四：AI 特有风险检测 🤖

基于 OWASP LLM Top 10 2025，逐项检测：

| 编号 | 风险 | 检测要点 |
|:----:|:----:|:--------:|
| LLM01 | 提示词注入 | 用户输入是否隔离、指令是否可被覆盖 |
| LLM02 | 敏感信息泄露 | System Prompt 是否含密钥 |
| LLM05 | 不当输出处理 | LLM 输出是否直接执行（eval/exec） |
| LLM06 | 过度代理授权 | allowed-tools 是否超过功能需求 |
| LLM09 | 错误信息 | 是否可能生成不安全的代码建议 |

### 维度五：合规性评估 📋

- **权限最小化**：allowed-tools 是否仅申请必需权限
- **功能一致性**：description 是否与实际能力一致
- **参赛合规**：SKILL.md 规范、命名规范、内容合规
- **安全责任**：数据使用政策、问题反馈渠道

## 📊 量化风险评分

```
风险总分 = 100 - Σ(风险扣分) + Σ(加分项)

🔴 致命风险：每项扣 15-25 分
🟠 高危风险：每项扣 8-14 分
🟡 中危风险：每项扣 3-7 分
🟢 低危风险：每项扣 1-2 分
✅ 加分项：有SBOM(+5)、有代码签名(+5)、最小权限(+3)

🟢 90-100 优秀 · 🟡 80-89 良好 · 🟠 60-79 中等 · 🔴 0-59 高风险
```

## 🔒 5项强制避坑筛查

每个被审计 Skill 必须通过以下5项检查，未通过直接标记风险：

| # | 检查项 | 过滤内容 |
|:-:|:------:|:--------:|
| ① | 致命命令检测 | rm -rf /、mkfs、dd覆写等破坏性操作 |
| ② | 凭证泄露检测 | 硬编码API Key、私钥、Token等敏感信息 |
| ③ | 恶意行为检测 | 挖矿代码、后门连接、数据窃取逻辑 |
| ④ | 混淆代码检测 | Base64/十六进制编码隐藏执行、变量名混淆 |
| ⑤ | 权限越界检测 | allowed-tools超出功能需求的工具授权 |

## 📦 安装方式

### 方式一：Git Clone

```bash
git clone https://gitee.com/zgdfsgddd/skill-security-auditor-skill.git
```

### 方式二：手动下载

1. 点击仓库右上角 **克隆/下载** → **下载 ZIP**
2. 解压到本地

### 导入 TRAE SOLO

1. 打开 TRAE SOLO 桌面端
2. 点击左侧 **Skills** 面板
3. 点击 **导入 Skill**
4. 选择 `skill-security-auditor-skill` 文件夹
5. 确认导入
